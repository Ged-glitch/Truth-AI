# Truth-AI Roadmap

This roadmap is intentionally short so contributors can see what matters next.
Detailed milestone discipline remains in `CODEX_HARNESS.md`.

## Now

- Harden the verified-chat adapter boundary.
- Keep Supabase-authenticated run persistence working through Vercel.
- Improve the static console so users can test the live workflow clearly.
- Add public standards library samples without copying paid standards text.

## Next

- Finish the deterministic backend milestones for graph validation, predicates,
  gate decisions and ledger continuity.
- Expand standards ingestion for authorised uploads and official source links.
- Add contributor-friendly fixtures for common truth-class failures.
- Add public issue labels for `good first issue`, `documentation`, `frontend`,
  `adapter`, `kernel` and `security`.

## Later

- Add calibrated advisory evaluator packs for DeepEval, Ragas and Open RAG Eval.
- Publish an evaluation report for injected-fault precision and replay stability.
- Add integration examples for agents that consume repair contracts.
- Package a stable public API surface for external apps.
